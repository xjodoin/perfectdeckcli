# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for security-sensitive problems.

Report vulnerabilities privately to:

- `xavier.jodoin@greybox.biz`

Include:

- A clear description of the issue
- Steps to reproduce
- The affected command, MCP tool, or integration
- Any relevant logs or example payloads, redacted as needed

## Response expectations

- Initial acknowledgment: within 5 business days
- Triage and impact assessment: as soon as practical
- Public disclosure: after a fix or mitigation is available

## Scope

Security reports are especially useful for:

- Credential handling issues
- Unsafe file reads or writes
- Unexpected network exposure through MCP tools
- Store API authentication or token handling problems
